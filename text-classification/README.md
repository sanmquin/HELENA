# Text Classification Evaluation

This directory contains a Python notebook for evaluating text classification accuracy using Gemini models and semantic embeddings.

## Workflow

1. **Data Loading**: Loads a JSON file from a Google Drive path containing categories and their respective texts.
2. **Data Splitting**: Splits data into 80% training and 20% testing per category.
3. **Description Generation**: Iteratively generates descriptions for each category using Gemini-3.1 Flash lite based on training data.
4. **Batch Classification**: Classifies training data in randomized batches of 20 using the generated descriptions.
5. **Performance Evaluation**: Analyzes accuracy at global and category levels with charts and tables.
6. **Semantic Analysis**: Measures how distances across categories are preserved using semantic embeddings.
7. **Correlation Analysis**: Determines the correlation between description accuracy and preserved semantic distance.
8. **Ablation Study**: Compares results with descriptions generated by Gemini 3.5 Flash in a single request.

## Directory Structure

- `classification_evaluation.ipynb`: The main notebook (optimized for Google Colab).
- `outputs/`: Directory for storing intermediate LLM responses and results (to avoid hitting quotas).

## Semantic Distance Metric

The metric evaluates the preservation of distances between categories:
1. Calculate the centroid embedding of texts for each category ($C_{text}$).
2. Calculate the embedding of the generated description for each category ($C_{desc}$).
3. Compute the cosine distance matrix between all $C_{text}$ pairs ($D_{text}$) and all $C_{desc}$ pairs ($D_{desc}$).
4. The preservation score is the Pearson correlation between the flattened upper triangles of $D_{text}$ and $D_{desc}$.
